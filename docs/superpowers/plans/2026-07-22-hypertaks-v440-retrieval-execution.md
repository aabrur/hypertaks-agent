# Hypertaks 4.4.0 Retrieval and Execution Implementation Plan

**Goal:** deliver a publishable cross-agent plugin update with retrieval
intelligence, stronger contracts, targeted capabilities, professional execution
standards, and evidence-backed visual delivery.

## Workstream 1: Baseline and design

- Preserve the certified 4.3.0 parent as the comparison point.
- Record the approved architecture and strict product boundary.
- Run the current validator, eval integrity check, static suite, and unit tests.

## Workstream 2: Retrieval and evidence

- Add the canonical Retrieval Intelligence Router.
- Define exact, semantic, mixed, metadata, fusion, reranking, evaluation, and
  fallback rules.
- Add evidence-pack and retrieval-metric output shapes.
- Integrate retrieval strategy into Phase 0 through Phase 5.

## Workstream 3: Contract integrity

- Add request, method, evidence, process, destination, validation, and signature
  fields to the machine-readable contract.
- Tighten activation rules for builds and external effects.
- Preserve zero-sized Nano and proportional Lite behavior.

## Workstream 4: Professional execution and visuals

- Add Python, Matplotlib, TypeScript, UI/UX, and image-generation standards.
- Add the Visual Necessity Router.
- Extend agent briefs and final deliverables with execution and visual evidence.

## Workstream 5: Runnable utilities

- Add a standard-library Python retrieval evaluator.
- Add a Matplotlib report generator with PNG and SVG output.
- Add a strict TypeScript routing reference and executable runtime tests.

## Workstream 6: Evaluation and validation

- Add EV-50 through EV-65.
- Extend structural validation for the new canonical fields and references.
- Add Python unit tests and TypeScript type and runtime checks.
- Keep behavioral status pending unless fresh independent runs exist.

## Workstream 7: Release synchronization

- Bump all live records to 4.4.0.
- Update README, changelog, checkpoint, release notes, skill card, eval docs,
  installation metadata, and CI.
- Regenerate figures only when source facts change.

## Workstream 8: Package and handoff

- Run the complete validation matrix.
- Produce a patch, Git bundle, source archive, implementation report, and
  publish-readiness report.
- Do not push, tag, deploy, or submit to a marketplace without a separate Boss
  approval.
